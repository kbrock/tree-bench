# Benchmark Results

PostgreSQL, Ruby 4.0.1, Rails 7.2, ~830 nodes across three tree shapes (wide, deep, mixed).

## Reports

### [Config comparison](read_bench_configs.md)

Compares ancestry serialization formats (mp1, mp2, mp3, ltree, array) and enhancements
(parent column, root column, virtual columns). Generated by `read_bench.rb --all`.

Each operation is measured with and without the `ancestor_ids` ivar cache to separate
parse time from cached access. Operations like `children` and `descendants` that use
AR associations include a "cached" variant showing the cost of association cache hits
vs cold access.

### [Write comparison](write_bench_configs.md)

Insert, move, and destroy operations across configs. Generated by `write_bench.rb --all`.

### [Cross-library comparison](compare_bench.md)

Same read operations measured across three setups:

- **ancestry** — mp3 format, scope-based access (no AR associations)
- **ancestry+assoc** — mp3 with virtual `parent_id` column, enabling `has_many :children` and `belongs_to :parent`
- **closure_tree** — hierarchy table with AR associations

The "cached" variants show how each library behaves on repeat access within a request.
The `4.descendants` and `4.preload(:children)` operations load multiple nodes and their
related data, showing where preloading via associations reduces query count.

Generated by `compare_bench.rb`.

## Regenerating

```bash
# run benchmarks (results accumulate in JSON files)
DB=pg bundle exec ruby read_bench.rb --all
DB=pg bundle exec ruby compare_bench.rb

# generate reports from JSON
bundle exec ruby markdown_report.rb
bundle exec ruby html_report.rb
```

## Interpreting the tables

Tables are grouped by tree shape and metric (ips, queries, rows).
Bold values indicate the best in each row. Rows where all values
are within error of each other are omitted.

For observations and analysis, see ../findings.md.
